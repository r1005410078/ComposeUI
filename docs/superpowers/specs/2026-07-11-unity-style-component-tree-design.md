# ComposeUI Unity 风格组件树设计

## 目标

组件树参考 Unity Hierarchy 的操作方式，用一棵可折叠的树展示当前画板的完整页面结构。它负责表达父子关系、同级顺序、节点状态和编辑上下文；空间位置和尺寸仍由画布负责，具体属性由属性面板负责。

组件树、画布和属性面板必须共享同一个选区和同一份 `PageDocument`，任何一侧发生选择或结构变更时，其他区域立即同步。

## 树结构

- 页面画板是组件树的固定根节点，不能被删除或拖入其他节点。
- 容器、自定义组件定义和允许子内容的业务组件可以包含子节点。
- 基础图形、文本、图片以及不支持插槽的业务组件是叶子节点。
- 节点可以按需展开和折叠；折叠只影响树的显示，不影响页面渲染。
- 父子关系表示 DOM、布局和生命周期归属，不只是视觉分组。
- 同一父节点下的顺序决定 DOM 顺序；自由布局发生视觉重叠时，也作为默认绘制顺序。

## 节点类型

组件树至少区分以下节点：

- 页面画板。
- 自由布局容器。
- Auto Layout 容器。
- Grid 容器。
- 框架无关基础节点，例如文本、图片、矩形和 SVG。
- 宿主注册的业务组件。
- 自定义组件定义。
- 自定义组件实例。
- Group 普通分组。
- 缺失组件、无效节点和导入降级节点。

每种节点使用稳定的类型图标。节点行还可以显示布局类型、业务组件框架、实例状态和错误状态，但不能用过多常驻文字挤压节点名称。

## 节点行信息

节点行从左到右包含：

1. 展开/折叠按钮，仅容器节点显示。
2. 节点类型图标。
3. 可编辑节点名称。
4. 必要的状态标识，例如 `Auto`、`Grid`、组件实例或缺失组件。
5. 锁定、隐藏、错误等状态图标。

名称默认来自组件显示名称，但用户可以单独重命名节点。重命名只修改编辑器名称，不修改组件注册 id。

节点名称过长时单行省略，并通过 Tooltip 显示完整名称。状态图标必须有可访问名称和 Tooltip。

## 选择与定位

- 单击节点执行单选，并在画布中显示对应选框。
- 使用平台标准修饰键增加或移除选区；使用 Shift 选择连续范围。
- 从画布选中节点时，组件树自动展开祖先并滚动到对应节点。
- 多选节点可以来自同一个父节点或不同父节点；仅在操作语义明确时允许批量结构操作。
- 选中隐藏节点时可以在树中编辑属性，但画布不渲染它的页面内容。
- 锁定节点仍可在组件树中选中和解锁，但不能通过画布拖动或调整尺寸。

## 拖拽与层级调整

拖拽节点时显示三种明确落点：

- 节点上方：插入为目标节点的前一个同级节点。
- 节点下方：插入为目标节点的后一个同级节点。
- 节点内部：成为目标容器的子节点。

拖拽过程中必须显示插入线或容器高亮，并在提交前校验目标是否接受该节点。禁止形成循环父子关系，禁止把节点放入叶子节点，禁止把不兼容框架的业务组件拖入当前画板。

放入不同布局容器时执行布局转换：

- 放入自由布局容器时，根据当前世界坐标计算新的局部位置和尺寸。
- 放入 Auto Layout 容器时，移除自由坐标，初始化排序、尺寸模式和对齐数据。
- 放入 Grid 容器时，根据落点计算 `x`、`y`、`w`、`h`，再执行碰撞和紧凑排列。

结构调整必须作为一个原子命令进入撤销/重做历史。多节点拖拽需要保持所选节点之间的原始相对顺序。

## 画布同步

- 组件树变更父子关系后，画布立即重新计算布局并更新 DOM。
- 画布拖动节点进入其他容器时，组件树实时显示候选父节点和插入位置。
- 画布与组件树共享悬停状态；从树中悬停节点时，可以在画布中显示轻量轮廓。
- 画布缩放和平移不会改变组件树状态。
- 树中同级顺序调整后，运行时 DOM 顺序必须与文档顺序一致。

## 显隐、锁定与错误状态

隐藏父节点会隐藏整棵子树，但不覆盖子节点自己的显隐配置。重新显示父节点后，子节点恢复各自状态。

锁定父节点会阻止其子树在画布中的直接变换，但仍允许在组件树中选择、检查和解锁。是否允许编辑子节点属性由权限策略决定。

缺失业务组件、无效数据绑定、不可用资源和 Figma 导入降级必须在对应节点显示状态。错误图标可以打开诊断信息，并定位到属性面板中的相关配置。

## 自定义组件与实例

自定义组件采用类似 Unity Prefab 的“定义 + 引用实例”模型。定义是唯一模板，实例保存引用和允许的覆盖，不复制整棵内部节点树。

### 自定义组件定义

`ComponentDefinition` 存放在 `PageDocument.componentDefinitions` 中，不直接作为页面根节点的普通子节点。它至少包含：

- 稳定的 `definitionId`、显示名称和所属 `runtime.adapterId`。
- 组件内部的根节点和完整节点子树。
- 对外公开的属性、插槽和事件契约。
- 当前修订号，用于诊断实例是否已同步到最新定义。
- 可选的分类、缩略图、描述和来源信息。

定义内部节点使用稳定的 `definitionNodeId`。修改节点名称、位置或属性时不得无故重建这些 id，否则实例覆盖无法继续定位目标。

定义只属于当前页面文档。跨页面、跨项目共享的组件库不进入核心版本；后续如需支持，应通过独立的组件库包、版本和迁移协议实现，不能直接引用另一个页面文档中的节点。

### 创建自定义组件

用户选择一个节点或多个具有共同父节点的连续节点后，执行“创建自定义组件”：

1. 校验选择中不存在当前画板不兼容的框架、无效节点或循环引用。
2. 多节点选择先创建一个内部根容器，并保留节点的视觉位置和同级顺序。
3. 将选中子树移动到新的 `ComponentDefinition`，为定义和内部节点生成稳定 id。
4. 在原位置插入一个 `ComponentInstance`，保持原有边界、布局位置和父节点顺序。
5. 打开组件契约编辑器，让用户选择需要公开的属性、事件和插槽。
6. 整个创建过程作为一个原子命令，可以一次撤销。

普通 Group 只用于组织和整体变换，不自动成为可复用组件。只有显式执行“创建自定义组件”后才生成定义和实例。

### 自定义组件实例

`ComponentInstance` 是页面节点树中的轻量引用节点，至少保存：

- 实例自身的 `nodeId`。
- 引用的 `definitionId`。
- 实例在父容器中的布局数据，例如自由坐标、Auto Layout 子项配置或 Grid 的 `x/y/w/h`。
- 已声明公开属性的实例覆盖值。
- 已声明插槽的实例内容。
- 实例事件到页面动作的绑定。

实例的外部布局属于实例本身，不属于组件定义。例如同一个卡片组件实例可以在不同 Grid 中拥有不同的跨度，但其内部标题、图标和内容布局仍由定义控制。

运行时通过 `definitionId` 解析定义树，再应用实例覆盖和插槽内容。实例不能持有定义内部节点树的完整副本，避免定义更新后产生不可控分叉。

### 公开属性

定义可以把内部节点的可编辑值声明为公开属性，例如标题文本、图标资源、背景色或业务组件 prop。公开属性包含稳定的 `propertyId`、显示名称、数据类型、默认值、校验规则和目标映射。

一个公开属性可以映射到一个或多个内部目标，例如 `titleColor` 同时修改标题和图标颜色。删除或改变公开属性类型时，必须检查所有实例覆盖并提供迁移或重置结果。

实例属性有三个来源，优先级从高到低为：

1. 实例上的数据绑定。
2. 实例静态覆盖值。
3. 定义中的默认值。

未被定义公开的内部属性不能在实例上下文中直接修改。需要修改时，用户必须进入定义编辑模式，或先将该属性公开。

### 插槽

定义可以把内部容器声明为插槽。插槽包含稳定的 `slotId`、名称、是否必填、允许节点类型、数量限制和默认内容策略。

实例插槽内容属于实例，但在渲染时挂载到定义指定的位置。插槽中的业务组件必须遵守当前画板的单框架约束。删除插槽前必须检查所有实例内容，要求用户选择取消、删除插槽内容或将内容移动到实例外部。

核心版本支持命名插槽和默认插槽，不支持任意修改定义内部非插槽节点的结构。

### 事件

定义可以公开语义事件，例如 `submit`、`change` 或 `openDetail`。事件可以来自定义内部的一个或多个业务组件事件，但实例只看到公开事件，不依赖内部实现。

每个实例可以将公开事件绑定到当前页面动作链。修改定义内部事件来源时，只要公开事件 id 不变，实例动作绑定就应保持有效。

### 实例覆盖

实例只能覆盖公开属性、插槽内容、公开事件绑定和实例外部布局。定义内部未公开的样式、结构、显隐和布局不能被单个实例覆盖。

组件树和属性面板必须区分：

- 未覆盖：使用定义默认值。
- 已覆盖：实例具有静态值或绑定。
- 定义已变更：默认值变化，但实例覆盖继续生效。
- 覆盖失效：对应属性、插槽或事件已从定义删除或改变类型。

用户可以重置单个覆盖、一个分类的覆盖或全部覆盖。重置操作必须可撤销。

### 定义编辑模式

双击实例或从上下文菜单选择“编辑组件定义”进入独立编辑上下文。画布只突出显示当前定义内容，页面其他节点弱化或隐藏，并通过面包屑显示“页面画板 / 组件定义名称”。

定义编辑期间：

- 修改内部结构和默认值会实时更新所有可见实例预览。
- 不允许修改某个实例的外部布局。
- 公开属性、插槽和事件通过专门的契约面板管理。
- 退出时无需生成实例副本，定义修改直接进入统一命令历史。

从某个实例进入定义编辑模式不代表只编辑该实例。界面必须明确提示修改会影响全部实例。

### 嵌套与循环引用

自定义组件定义可以引用其他自定义组件实例，以支持基础组件组合成复杂组件。系统必须构建定义依赖图，并禁止直接或间接循环引用：

```text
A → B → C → A   // 禁止
```

删除、复制、导入或迁移定义后都必须重新校验依赖图。渲染时还应设置最大递归深度作为异常数据保护，但不能依赖该限制替代循环校验。

### 解除实例关联

“解除实例关联”会把实例解析为当前定义节点树的普通副本，并应用当前属性覆盖和插槽内容。解除后，新节点不再跟随定义更新。

该操作必须：

- 为展开后的所有节点生成新的页面节点 id。
- 保持实例当前外部布局、视觉结果和同级顺序。
- 移除仅对实例有意义的覆盖和定义引用。
- 作为一个原子命令支持撤销。

### 删除定义

删除定义前必须检查所有直接和间接实例。存在引用时提供以下选择：

- 取消删除。
- 删除定义及全部实例。
- 将全部实例解除关联后删除定义。

核心版本不允许留下悬空 `definitionId`。如果导入或文档损坏产生悬空实例，组件树显示“缺失自定义组件”错误节点，并允许删除或重新绑定定义。

### 复制与粘贴

在同一文档中复制实例时，新实例继续引用原定义，只复制实例覆盖、插槽内容和事件绑定。

复制定义时创建新的 `definitionId` 和内部节点 id，并复制其依赖关系。复制到另一个文档时，需要连同依赖定义和资源一起导入；发生 id 冲突时重建本地 id，并生成导入报告。

### 组件树展示

- 定义和实例使用不同图标。
- 实例名称默认继承定义名称，但允许设置实例显示名称。
- 具有覆盖的实例显示覆盖标识；存在失效覆盖时显示警告。
- 实例节点默认可展开查看解析后的内部结构，但内部节点在实例上下文中为只读。
- 插槽内容在实例节点下以明确的插槽分组显示，并允许选择和编辑。
- 从内部只读节点执行编辑操作时，引导用户进入定义编辑模式或编辑对应的公开属性。

## 搜索与过滤

组件树支持按节点名称和组件注册 id 搜索，并可以按以下条件过滤：

- 节点类型或布局类型。
- 当前业务组件框架。
- 隐藏或锁定状态。
- 自定义组件实例。
- 缺失组件、无效绑定和其他错误状态。

搜索结果必须保留必要的祖先路径，避免结果脱离上下文。清除搜索后恢复用户此前的展开状态。

## 上下文菜单与命令

右键菜单根据节点类型和当前选区提供适用命令：

- 新建子节点或容器。
- 重命名、复制、粘贴、重复和删除。
- 组合、取消组合、创建自定义组件和解除实例关联。
- 移到最前、前移、后移和移到最后。
- 显示/隐藏、锁定/解锁。
- 展开子树、折叠子树和在画布中定位。

所有菜单命令必须调用统一命令系统，不允许组件树直接修改文档状态。

## 键盘操作与可访问性

- 方向键移动树焦点；左右键展开、折叠或进入父子层级。
- Enter 开始重命名，Escape 取消当前操作。
- Delete/Backspace 删除选中节点，并遵循宿主平台习惯和确认规则。
- 复制、粘贴、重复、撤销和重做使用平台标准快捷键。
- 节点树使用正确的 tree/treeitem 语义，状态、层级、展开状态和选中状态可被辅助技术识别。

快捷键只在当前编辑器实例获得焦点时生效，不得影响宿主页面或其他编辑器实例。

## 性能要求

- 树应支持数千节点，长列表采用虚拟化渲染。
- 展开、折叠、滚动和选择不能触发整棵文档树的重新渲染。
- 节点状态通过稳定 id 和细粒度订阅更新。
- 批量结构操作在提交后统一更新布局和选区，避免逐节点闪动。

## 验收场景

1. 用户从组件面板拖入业务组件，组件树和画布同时出现节点并保持选中。
2. 用户将节点从自由布局容器拖入 Auto Layout，节点树父子关系和布局数据正确转换。
3. 用户将多个节点拖入 Grid，原顺序保持，Grid 正确计算位置并处理碰撞。
4. 用户在画布选中深层节点，组件树自动展开、定位并高亮对应节点。
5. 用户锁定父容器后无法在画布移动子节点，但可以从组件树解锁。
6. 用户创建自定义组件并修改定义，多个实例同步更新且实例覆盖保持有效。
7. 缺失组件或无效绑定在树中显示错误状态，并能打开对应诊断。
8. 同一页面存在多个编辑器实例时，选择、快捷键和拖拽状态互不影响。
