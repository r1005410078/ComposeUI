# 操作日志 Output 工具栏优化设计

## 背景

当前 Output 面板将日志浏览、筛选、数据管理和回放控制全部平铺在同一行工具栏中。默认状态同时显示清空视图、四个状态筛选、六个分类筛选、自动滚动、导入、导出、启动回放和六个回放控制按钮。

这些按钮大量复用相似图标，且不区分当前是否选中日志、是否处于回放状态。用户无法仅凭视觉判断按钮用途，窄面板中还会出现拥挤和溢出。

## 目标

- 默认以日志查看和问题排查为主要任务。
- 保留现有查询、导入导出和隔离回放能力。
- 根据当前选择和回放状态只展示有效操作。
- 通过文字、状态和分组降低图标猜测成本。
- 适应 Dockview 面板宽度变化，不产生横向工具栏滚动。
- 沿用现有主题 token，不引入固定主题色。

## 非目标

- 不修改操作日志事件格式、存储格式或查询语义。
- 不改变 ReplayEngine 的确定性回放机制。
- 不增加新的日志分类或状态。
- 不在本次优化中重构日志详情的数据内容。

## 方案选择

采用上下文工具栏。相比固定展示全部功能，它能显著减少默认控件数量；相比将所有功能放入单个命令菜单，它能保留搜索、筛选和回放等核心操作的可发现性。

工具栏分为日志浏览、日志选中和回放三种状态。数据管理属于低频操作，收纳到“更多”菜单；状态和分类筛选合并到单个筛选弹层；回放控制只在对应状态出现。

## 状态模型

### 日志浏览状态

默认工具栏只显示：

```text
[搜索操作日志                         ] [筛选] [自动滚动] [更多]
```

- 搜索输入即时更新日志查询。
- “筛选”打开状态与分类筛选弹层。
- “自动滚动”明确显示开启或关闭状态。
- “更多”包含导入日志、导出日志和清空当前视图。
- 点击日志行会选中该事件并显示操作详情。

### 日志选中状态

选中支持回放的日志后，工具栏增加主操作：

```text
[搜索操作日志                  ] [筛选] [回放到此处] [更多]
```

- “回放到此处”使用播放图标和文字，不使用无标签图标。
- 不支持回放的事件不显示回放入口。
- 再次点击当前日志只保持选中，不隐式开始回放。
- 取消选择后恢复日志浏览状态。

### 回放状态

回放开始后，在普通浏览工具栏下方显示独立回放控制条：

```text
回放至 #46  当前 #18  状态：已暂停  一致
[上一步] [下一步] [运行到选中项] [验证]                         [停止]
```

- “上一步”和“下一步”执行单步回放。
- “运行到选中项”仅在选择了有效目标时启用。
- “验证”比较当前回放状态与记录状态。
- 检测到差异时显示差异摘要，并按需显示“继续回放”。
- “继续回放”只在非确定性差异发生后出现。
- “停止”使用停止图标和文字，退出隔离回放，不使用垃圾桶图标。
- 停止回放不会删除日志，也不会修改正式项目文档。
- 回放期间继续禁用 Workspace 顶部的运行和保存操作。

## 筛选交互

原有四个状态按钮和六个分类按钮合并到一个筛选弹层：

```text
状态
☑ 记录  ☑ 开始  ☑ 成功  ☑ 失败

分类
☑ 文档  ☑ 历史  ☑ 会话
☑ 工作区  ☑ 诊断  ☑ 系统

[重置]                         [完成]
```

- 状态和分类均支持多选。
- 没有启用筛选时显示“筛选”。
- 启用筛选后显示数量，例如“筛选 2”。
- 关闭弹层不会清除当前筛选条件。
- “重置”恢复全部状态和分类可见。
- 查询无结果时显示“没有符合条件的日志”和“重置筛选”。

## 更多菜单

“更多”菜单包含以下低频操作：

- 导入日志。
- 导出日志。
- 自动滚动开关；中等宽度下从主工具栏移入菜单。
- 清空当前视图。

“清空当前视图”必须明确说明只隐藏当前已显示的日志，不删除 IndexedDB 中的持久化事件。执行前需要二次确认。该操作不能与“停止回放”共用图标或文案。

导入的日志作为独立会话查看，不与当前实时日志直接混合。导入失败不得清空或替换当前日志。

## 视觉规范

- 继续使用现有主题色、间距、圆角、边框和状态 token。
- 工具栏高度与画布工具栏保持一致。
- 搜索框占据剩余空间，并设置可收缩宽度。
- 筛选、自动滚动和主要回放操作使用图标加短文字。
- “更多”使用省略号图标，并提供悬浮提示和无障碍名称。
- “回放到此处”使用现有强调色，作为选中状态下的主操作。
- 危险色只用于更多菜单中的确认步骤，不用于普通工具栏背景。
- 激活、悬浮、焦点和禁用状态必须有明显差异。
- 不使用多个外观相同的播放、勾选或漏斗图标表达不同命令。

## 响应式行为

### 宽面板

```text
[搜索……                    ] [筛选 2] [自动滚动：开] [回放到此处] [更多]
```

### 中等宽度

```text
[搜索……             ] [筛选 2] [回放到此处] [更多]
```

自动滚动移入更多菜单。

### 窄面板

```text
[搜索……                         ] [筛选 2] [更多]
```

“回放到此处”移到选中日志的详情区域。工具栏不得产生横向滚动、文字重叠或按钮挤压变形。

响应式切换基于 Output 面板自身可用宽度，而不是浏览器视口宽度，以适应 Dockview 拖拽调整。

## 错误与反馈

- 查询失败时在工具栏下方显示错误信息和“重试”。
- 导入失败时显示具体原因，并保留当前日志、筛选和选择。
- 导出失败时保留当前筛选和选择。
- 回放失败时停止不可继续的控制流程，并保留错误与差异详情。
- 筛选无结果属于空状态，不作为错误展示。
- 所有异步操作需要防止重复触发，并在执行期间呈现禁用或进行中状态。

## 可访问性与键盘操作

- 每个图标按钮必须具有准确的 `aria-label` 和悬浮提示。
- 筛选弹层和更多菜单支持键盘打开、方向键导航、Escape 关闭和焦点恢复。
- 搜索、日志选择、筛选和回放控制均可通过键盘完成。
- 筛选开关、自动滚动和当前选择使用标准可访问状态属性。
- 回放状态和异步错误通过合适的 live region 通知辅助技术。

## 实现边界

- 保留现有 `OperationLogControllerPort` 和 `ReplayControllerPort` 能力。
- Output 面板负责把控制器状态映射为浏览、选中和回放 UI 状态。
- 筛选弹层、更多菜单和回放控制条应拆成内部独立渲染单元，避免继续扩张单个挂载函数。
- 响应式状态使用容器查询或 `ResizeObserver` 驱动的面板尺寸状态，不依赖全局窗口宽度。
- 所有样式继续通过 Workspace CSS 和主题 token 表达。

## 验收标准

- 未选中日志且未回放时，主工具栏最多显示搜索、筛选、自动滚动和更多四项。
- 状态和分类筛选不再以十个独立图标平铺。
- 只有选中可回放日志后才显示“回放到此处”。
- 只有回放激活后才显示单步、验证、继续和停止等回放控制。
- “继续回放”只在发生非确定性差异后显示。
- 清空视图、停止回放和删除类动作具有不同图标和明确文案。
- 筛选数量、重置和组合查询行为正确。
- 中等与窄面板宽度下没有横向溢出或控件重叠。
- 导入、导出、查询和回放失败均提供可见反馈。
- 现有日志列表、详情、查询、导入导出和隔离回放能力保持兼容。
- 自动化测试覆盖三种工具栏状态、筛选弹层、更多菜单、响应式切换、键盘操作和错误状态。
