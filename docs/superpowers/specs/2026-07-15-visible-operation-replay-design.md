# 可见操作回放与日志滚动设计

**日期：** 2026-07-15  
**状态：** 已确认

## 背景

当前回放引擎在隔离的内存 Editor、Session 和 Workspace 中执行操作，只向输出面板返回序号、状态和一致性结果。隔离执行保证原始项目不受影响，但隔离状态没有投影到画布，因此用户看不到节点移动等 UI 变化。

输出日志列表默认开启自动滚动。点击日志行会重新渲染列表，而渲染逻辑在自动滚动开启时无条件滚动到底部，导致用户选中较早日志后立即跳到最后几条。

## 目标

- 回放过程在当前画布位置可见，但不修改原始项目。
- 点击“回放到此处”后，从最近检查点自动逐条播放到目标日志。
- 支持暂停、继续、上一步、下一步和停止。
- 选中历史日志后保持列表位置稳定，新日志不能抢走阅读位置。
- 回放差异和错误停留在对应帧，便于排查。

## 非目标

- 不直接使用回放操作修改当前 Editor。
- 不实现原始画布与回放画布的左右对比视图。
- 不回放网络、文件下载等外部副作用。
- 本阶段不提供可配置播放速度，固定使用约 300ms 的步骤间隔。

## 架构

### 隔离回放引擎

`ReplayEngine` 继续创建隔离的 Editor、Session 和 Workspace。所有处理器保持 `sideEffects: "disabled"`，原始项目和宿主环境不参与回放写入。

引擎每次执行操作后返回 `ReplayState`：

- 当前序号；
- 规范化文档快照；
- Session 状态；
- Workspace 状态。

### 回放控制器

`ReplayControllerState` 增加当前可展示的回放帧。Controller 负责：

- 从最近检查点启动；
- 逐次调用引擎 `step()`；
- 每约 300ms 发布一帧；
- 处理暂停、继续、前进、后退和停止；
- 在停止或启动失败时清除回放帧；
- 保留确定性状态和首个差异信息。

自动播放是 Controller 行为，不改变 ReplayEngine 的纯隔离职责。

### 画布预览

画布挂载接口增加只读预览能力。正常状态下，画布继续订阅原始 Editor 和 Session；回放激活后，画布使用 Controller 发布的 `ReplayState` 作为视觉数据源。

回放预览具备以下约束：

- 只替换画布渲染所需的文档、选择和视口数据；
- 禁用节点拖动、缩放、重命名和其他编辑命令；
- 显示“回放模式 · 当前 #n / 目标 #m”状态标识；
- 停止后清除预览并重新显示原始 Editor 状态；
- 不向操作日志写入预览内部状态变化。

## 回放交互

1. 用户点击一条日志，该日志成为回放目标。
2. 用户点击“回放到此处”。
3. Controller 加载 bundle，并从目标序号之前最近的检查点创建隔离运行时。
4. 画布进入只读回放模式，首先显示检查点帧。
5. Controller 以约 300ms 的间隔逐条执行到目标序号，每一步更新画布和当前日志高亮。
6. 到达目标后保留目标帧，用户可以继续使用上一步、下一步和验证。
7. 用户点击停止后，画布恢复原始项目和编辑能力。

拖动、缩放等连续交互以已经记录的成功事务为一个可见步骤，不重放原始指针采样。

## 日志滚动

日志列表区分“跟随最新日志”和“阅读历史”两种状态：

- 默认跟随最新日志并滚动到底部。
- 用户点击任意日志行后，立即进入阅读历史状态，保持当前 `scrollTop` 和选中行位置。
- 新日志继续写入并更新数据，但不改变滚动位置。
- 阅读期间显示“新增 N 条”入口。
- 用户点击“回到底部”后滚动到最新日志、清零新增计数，并重新开启自动跟随。
- 当前回放步骤高亮不能触发列表自动滚动。

工具栏中的自动滚动开关继续保留。用户手动关闭时，不自动恢复；点击“回到底部”明确恢复自动跟随。

## 错误处理

- bundle 导入、完整性校验或回放运行时创建失败：退出回放模式，保留原画布并在输出面板显示错误。
- 操作处理器产生差异：暂停在差异帧，保持画布预览，显示序号、差异类型、预期值和实际值。
- 无法生成回放帧：暂停并显示状态错误，不用部分帧覆盖画布。
- 用户停止或组件销毁：取消未完成的播放定时器，忽略迟到结果并清除预览。

## 测试

### 单元测试

- ReplayController 启动后逐帧发布 `ReplayState`。
- 暂停取消后续定时步骤，继续从当前序号恢复。
- 上一步重建到前一序号，下一步只推进一个操作。
- 停止和启动失败清除帧并阻止迟到结果覆盖状态。
- 画布收到回放帧后显示隔离文档并禁用编辑，清除后恢复原始文档。
- 点击日志行保持 `scrollTop`，后续日志增加计数但不滚动。
- 点击“回到底部”恢复自动跟随并清零计数。

### 集成与 E2E

- 拖动节点并持久化日志。
- 刷新页面后选择移动成功日志并开始回放。
- 断言画布从检查点位置逐步移动到目标位置。
- 断言原始 Editor 文档在回放期间保持不变。
- 断言选中历史日志后列表不跳到底部。
- 停止回放后断言画布恢复原始状态和编辑能力。

## 验收标准

- 用户能直接看到回放导致的节点位置和状态变化。
- 回放不会修改原始文档、历史栈或产生回放内部操作日志。
- 选中任意历史日志后列表位置保持稳定。
- 回放到目标、单步、暂停、继续、停止和差异停留都有自动化测试覆盖。
